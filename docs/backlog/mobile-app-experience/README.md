# Mobile App Experience

## Feature Goal

Make the product behave and feel like a proper mobile app on a phone, not just a website in a browser.

## Why This Matters

This app is meant for frequent, real-world use on a phone, so the experience should feel installable, fast, and smooth in the ways that matter on mobile.

## What "Mobile App Experience" Means Here

- The app can be installed to the phone home screen
- The UI feels native enough to use like an app
- The shell, spacing, and interactions work well on modern phones
- The experience stays dark-mode only and mobile-first
- Weak or unstable network conditions are handled reasonably well

## V1 Scope

- PWA foundation
- Installable app behavior
- Safe area and viewport handling
- App shell and navigation model
- Touch interaction polish
- Branding assets and metadata
- Basic offline and poor-network behavior

## Out of Scope

- Multi-user behavior
- Complex offline sync
- Native app store packaging
- Push notifications
- Background sync systems
- Desktop-specific shell patterns
