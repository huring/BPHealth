# Card 05: Automatic Tags

Status: done

## Objective
Define system-generated tags.

## Scope
- Automatically add a time-of-day tag based on when a measurement is created:
  - morning
  - lunch
  - evening
- No manual selector needed
- Logic can be approximate based on local device time

## Acceptance Criteria
- Correct tag is auto-applied
- User does not need to think about time-of-day tagging
- System remains simple and predictable

## Notes
- Automatic tags should reduce friction, not add new maintenance work.
