# Card 04: Measurement Integration

Status: done

## Objective
Integrate tags into the measurement flow.

## Scope
- Tags are added when creating a measurement
- Tags are displayed as chips attached to the measurement
- Sys and dia labels must remain fully written, not abbreviated
- Date defaults to today
- Date should be editable via a lightweight interaction, not a full input field

## Acceptance Criteria
- Tagging fits naturally into the measurement flow
- No added friction to core BP logging
- UI remains clean and focused

## Notes
- Keep the measurement flow fast enough for quick mobile entry.
