# Card 02: Selection and Interaction

## Objective
Update the current selection display behaviour

## Scope
- Update positioning of value display for current selection
- Selecting an already selected value should un-select and revert to default
- selected value should display as a chip on top of the vertical bar that is selected
- The chip should have a red background and white text
- The "selected" header is not necessary
- The selected chip should only display the value and the date + time
- The current selection should also be a red
- The red color for the column and the chip should complement the other colors in the app

## Acceptance Criteria
- Selection feels responsive on touch
- Selected state is clear on the range bars
- Accessibility behavior remains intact

## Status
- Done

## Notes
- Selection should emphasize the whole vertical range, not just the endpoints
