# Card 03: Mobile Viewport and Safe Areas

## Objective

Make the UI behave correctly on phone screens.

## Scope

- Viewport sizing
- Safe area insets
- Bottom areas, top notch areas, and edge spacing
- Prevent awkward scrolling or browser chrome issues

## Acceptance Criteria

- The app layout feels correct on modern phones
- No broken spacing near screen edges
- Mobile screen behavior feels intentional

## Notes

- Prioritize real phone ergonomics over desktop assumptions
