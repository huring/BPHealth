# Card 04: App Shell and Navigation

## Objective

Define an app-like shell and navigation model.

## Scope

- Mobile header behavior
- Bottom navigation if appropriate
- Persistent structure for key views
- Reduce "website" feel

## Acceptance Criteria

- The app structure feels like a mobile product
- Key actions are easy to reach with one hand
- The shell is simple and not overbuilt

## Notes

- Keep navigation minimal and focused on the core workflow
