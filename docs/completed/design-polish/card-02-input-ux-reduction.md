# Card 02: Input UX Reduction

Status: done

## Objective

Reduce reliance on traditional forms.

## Scope

- Identify where the current UI can feel less form-heavy
- Prefer simpler and faster input patterns
- Consider segmented controls, toggles, chips, quick actions, step-based entry, or bottom-sheet style interactions where appropriate

## Acceptance Criteria

- Input flows are faster and more intuitive
- Unnecessary fields or friction are reduced
- Interaction patterns feel lightweight and mobile-friendly

## Notes

- Implemented in [`app/page.tsx`](../../../app/page.tsx) and [`app/globals.css`](../../../app/globals.css)
- Keep the changes practical and targeted to high-friction spots
