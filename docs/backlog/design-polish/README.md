# Design Polish

## Feature Goal

Improve the product feel and usability so the app feels more modern, intuitive, and mobile-first.

## Why This Matters

The core features are in place, so the next step is to make the experience feel calmer, faster, and better suited to daily use.

## Mobile-Only Design Principle

The app is intended for mobile use only, so the design should prioritize phone screens, thumb-friendly interactions, and small-screen hierarchy.

## Dark-Mode-Only Principle

Dark mode is the only intended theme. Keep the UI dark by default and avoid light colors unless they are needed for contrast, readability, or chart clarity.

## V1 Scope

- Mobile-first layout improvements
- Faster and less form-heavy input patterns
- A more polished dark visual style
- Dashboard hierarchy and scanability improvements

## Out of Scope

- New product features
- Desktop-specific layouts
- Light mode support
- Major navigation rewrites
- Advanced animation systems
